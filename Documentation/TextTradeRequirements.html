Introduction

Our project is an application named Text Trade that is basically a textbook marketplace and allows Austin Peay students to come together and buy or sell their used textbooks. Our team plans to keep the application simple and easy to use. There will be two essential GUIs: one for users, and one for moderators. Each user has to register an account in order to start trading on Text Trade. The user then can create a list of books he wants to buy and/or sells. Each textbook entry must include all the necessary information such as title, author, edition, ISBN, associated course(s) that the textbook is used for, and book condition. A user can search for a book in the marketplace, find potential buyers, communicate with other traders, and report misdeeds to moderator. To keep things neat, we plan to implement the search algorithm based on ISBN numbers and course numbers. User Requirements Definition

  • Buyer

    1. Name: view_Listings
      Type: Functional
      Summary: Buyers must be able to view listings

    2. Name: manage_Schedule
      Type: Functional
      Summary: Buyers shoudl be able to add and drop classes from their schedule

  • Seller

    1. Name: manipulate_Listings
      Type: Functional
      Summary: Sellers must be able to add, remove, or change listings

  • Moderator

    1. Name: moderator_Powers
      Type: Functional
      Summary: Moderators must be able to manipulate the table, message users, and ban users

  • System

    1. Name: search_Table
      Type: Functional
      Summary: Application must allow for users to search through the table

    2. Name: create_Account
      Type: Functional
      Summary: Users must create accounts to store personal info

    3. Name: edit_profile
      Type: Functional
      Summary: Users should complete their profile page, filling out fields such as full name, DOB, phone number, campus email address, major, concentration, etc.

    4. Name: report_user
      Type: Functional
      Summary: Users should be able to report inapproriate activities to Moderators.

System Requirements Specifications

  • Buyer

    1. Name: save_listing
      Type: Functional
      Summary: Buyers shall be able to save listings of interest into their watch list

    2. Name: sort_results
      Type: Functional
      Summary: Buyers shall be able to sort results based on criteria such as "Cheapest", "Most Recent", etc.

    3. Name: filter_results
      Type: Functional
      Summary: Buyers shall be able to filter results based on criteria such as "Price", "Condition", etc.

    4. Name: message_user
      Type: Functional
      Summary: Users shall be able to communicate via internal message system

  • Seller

    1. Name: add_Listing_Seller
      Type: Functional
      Summary: Sellers must be able to put up listings

    2. Name: remove_Listing_Seller
      Type: Functional
      Summary: Sellers must be able to remove their own listings

    3. Name: edit_Listing_Seller
      Type: Functional
      Summary: Sellers shall be able to edit their own listings info.

  • Moderator

    1. Name: remove_Listing_Mod
      Type: Functional
      Summary: Moderators must be able to remove listings

    2. Name: remove_user_Mod
      Type: Functional
      Summary: Moderators shall be able to ban users who commit foul play

  • System

    1. Name: listing_Table
      Type: Functional
      Summary: Database must hold a table of the listings created by sellers

    2. Name: login_Storage
      Type: Risk
      Summary: login information must be encrypted

    3. Name: password_Requirements
      Type: Constraint
      Summary: Passwords must contain atleast 1 uppercase, 1 lowercase, be atleast 6 characters long, and 1 number
